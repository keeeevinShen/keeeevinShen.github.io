Leetcode backtrack pattern reflection

July 18, 2025
Algorithms LeetCode Problem Solving

Actually leetcode pattern is key to solving problems. I came across the backtracking pattern a few times and initially without recognizing the pattern, I struggled a lot. But once I know this pattern and I know how to think about it, solving this type of problem become straightforward.

The key insight is: CHOOSE-EXPLORE-UNCHOOSE.

But I think this phrasing doesn't capture the most important part.

For me, the key insight is you need to think about the state you need to track. what is the information I need to carry when I travel. Once you figure it out, implementing these problems become much cleaner. In fact, by using this approach, I can just go to the medium difficulty without even going through the easy level. Because the approach is so streamline.

1: think about what state we need to track at each position. (normally it is a current solution so far AND the remaining options) and you need to be very precise at what these mean. for example is the remaining options everything or is it something else.

2: Think about what are all the possible next choice at current position

3: what happens after the next choice, i.e. what is the new state

4: Make the base case

The question when you set the basecase is when should I stop exploring this path.

Think about what state will make the path end. That is our base case.

And the solution is almost always following this structure, write it down first so we have more brain capacity to think about the detailed implementation.
